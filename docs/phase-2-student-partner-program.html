RichDeals Student Partner Program

Turn students into earners — they share deals, earn a % of revenue, and watch it grow on a live dashboard.

The idea in one line

A student signs up, generates a personal affiliate link for any product, shares it, and earns a share of the commission when friends buy — with a dashboard showing Pending → Confirmed → Paid earnings, gated by the return period. Free, viral, pay-only-on-sales distribution. (Proven model: CashKaro, EarnKaro, GoPaisa. Our edge = curated deals + student niche + clean UX.)

How a student earns

  1. Sign up with Google → gets a unique ref code.
  2. Pick any deal → personal link richdeals.in/out/<dealId>?ref=<studentId>.
  3. Shares on WhatsApp / Instagram / college groups.
  4. Friend clicks → we log it → 302 to store with our affiliate tag.
  5. Friend buys → commission tracked against the student's ref.
  6. Dashboard updates: Clicks → Pending ₹ → Confirmed ₹ → Paid ₹.
  7. UPI payout once confirmed earnings cross a minimum (e.g. ₹250).

Earnings lifecycle

👆 Click→ ⏳ Pending (in return window)→ ✅ Confirmed (window closed)→ 💸 Paid (UPI)

Commission confirms only after the 7–30 day return window + the network's monthly cycle. The dashboard shows the pending pool with an expected-confirm date — honest expectations, fewer support tickets. This is the "kitna milega after return period" live showcase.

Revenue model

RichDeals earns ~1–8% net from Amazon/Flipkart. Student gets a share of NET — suggest 30–50%, not gross.

⚠️ Compliance — the make-or-break

Raw Amazon Associates ToS generally bans paying sub-affiliates / incentivised sharing — doing it on a raw Amazon tag risks a permanent account ban.

Fix: route all student links through EarnKaro / Cuelinks — they ARE sub-affiliate networks built for multi-user attribution + payouts + compliance. Build ON TOP of them, never raw Amazon. Flipkart: same via the network.

MVP — smallest thing that earns

  • Google login for students.
  • ref on the existing /out redirect → Click row stores studentId.
  • One dashboard: clicks, pending ₹, confirmed ₹, paid ₹, share link.
  • Manual monthly reconciliation (upload network report → credit) + manual UPI payout.

Defer: auto-payout, leaderboards, tiers, custom short links, mobile app.

Verdict

Strong growth lever — do it in Phase 2, but build on EarnKaro/Cuelinks (compliance + payout), not raw Amazon. Start with the manual MVP, prove the click → sale → confirm loop, then automate.
